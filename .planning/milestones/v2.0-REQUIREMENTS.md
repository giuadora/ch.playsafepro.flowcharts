# Requirements: PSP Inspektions-Flowchart

**Defined:** 2026-02-12
**Core Value:** Give playground owners a clear, norm-based understanding of inspection obligations per SN EN 1176-7, so they can fulfill their duty of care (Werkeigentumerhaftung, Art. 58 OR)

## v2.0 Requirements

Requirements for v2.0 Flowchart Split. Each maps to roadmap phases.

### Flowchart Split

- [x] **SPLIT-01**: Single flowchart split into two separate draw.io files with matching SVG, PDF, and Mermaid exports
- [x] **SPLIT-02**: FC1 (Neuer Spielplatz) starts at "Neuer Spielplatz" and ends at "Inspektionszyklus beginnen"
- [x] **SPLIT-03**: FC2 (Bestehender Spielplatz) starts at "Inspektionszyklus beginnen" and loops back via "Naechster Inspektionszyklus"
- [x] **SPLIT-04**: Both flowcharts have a subline below the main title — "Neuer Spielplatz" on FC1, "Bestehender Spielplatz" on FC2

### Node Modifications

- [x] **NODE-01**: FC1 "Inspektion nach Installation" node colored green (inspection activity)
- [x] **NODE-02**: FC1 "Inspektionsplan erstellen" rendered as subprocess symbol (double-bordered rectangle)
- [x] **NODE-03**: FC2 new "Bericht archivieren" node as green rectangle (Action) between Inspektionsbericht erstellen and Instandhaltungs-Management
- [x] **NODE-04**: FC2 inspection types renamed to SN EN 1176-7 terminology: Visuelle Routine-Inspektion, Operative Inspektion, Jaehrliche Hauptinspektion

### References & Footnotes

- [x] **REF-01**: All nodes referencing a norm clause carry a footnote superscript
- [x] **REF-02**: "Inspektionsplan erstellen" audited against SN EN 1176-7 and footnoted if applicable
- [x] **REF-03**: Each flowchart has its own complete footnote box (only footnotes relevant to that flow)

### Legend & Documentation

- [x] **DOC-01**: Subprocess symbol added to legend in both flowcharts
- [x] **DOC-02**: README updated to describe both flowcharts (Neuer Spielplatz + Bestehender Spielplatz)
- [x] **DOC-03**: Root README updated with new file listing

### File Management & QR

- [x] **FILE-01**: Files renamed to clearly distinguish the two flows (Neuer-Spielplatz.* and Bestehender-Spielplatz.*)
- [x] **FILE-02**: Each flowchart's "Als PDF herunterladen" QR points to its own PDF
- [x] **FILE-03**: Both flowcharts' "Als Vorlage herunterladen" QR points to the shared README

### Quality Assurance

- [x] **QA-01**: Both flowcharts pass center alignment verification (vertical + horizontal)
- [x] **QA-02**: All arrows connected — no dangling edges
- [x] **QA-03**: Margins maintained on all edges (no content touching page bounds)
- [x] **QA-04**: No node overlaps
- [x] **QA-05**: Footer with 3-column corporate layout present on both flowcharts
- [x] **QA-06**: PSP logo and branding present on both flowcharts
